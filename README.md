# Email Support Classifier

Проект обрабатывает набор входных писем, извлекает из них структуру, определяет категорию и сохраняет результаты каждого запуска в отдельную папку.

## Что делает программа

Программа:

1. читает письма из `data/input`
2. парсит тему, отправителя и тело письма
3. определяет категорию письма по набору правил
4. копирует письмо в папку соответствующей категории
5. сохраняет лог и итоговый отчет для конкретного запуска

Важно:
- исходные файлы в `data/input` не удаляются
- каждый запуск сохраняется отдельно в `data/output/<run_id>/`

## Категории

- `critical_incidents` — критические и массовые сбои
- `access_requests` — запросы на доступы, права, логины, пароли
- `technical_issues` — локальные технические проблемы
- `monitoring` — автоматические уведомления мониторинга
- `service_requests` — запросы помощи по сервису, порталу, API, личному кабинету
- `internal_communication` — внутренняя рабочая переписка, созвоны, дайджесты, плановые техработы
- `non_support` — письма не про IT-support: документы, больничные, отпуска, правки
- `spam_phishing` — спам и фишинг
- `unknown` — письмо удалось прочитать, но признаков недостаточно
- `unreadable` — файл невозможно корректно обработать как письмо
